Automation

Control how Overcoat processes posters and when its scheduled task runs.

Behaviour

Only reprocess items that changed since the last run (much faster). Off reprocesses everything.
Test mode — works out what it would do but doesn't change any posters.

Schedule

When the overlay job runs automatically. Pick a quiet hour — on a large library the run can take a while.

On, Overcoat keeps the task on the daily time below. Off, it stops touching the schedule and you manage it yourself in Dashboard → Scheduled Tasks (intervals, several times a day, on startup). Turning this off does not stop the task running — any trigger already set there stays active. To stop automatic runs entirely, remove the triggers in Dashboard → Scheduled Tasks.
A Jellyfin library scan re-adopts posters stored in your media folders and strips Overcoat's overlays off the affected items — Jellyfin does this automatically and it can't be turned off. With this on, Overcoat notices a scan finishing and re-applies within about a minute. It only re-does what the scan changed, and never touches your media-folder files. Recommended on.

Run & restore

These actions save the settings on every tab before starting their scheduled task.

Apply overlays

Process enabled libraries now using the settings currently shown on this page.

Restore originals

Put back clean posters saved in Overcoat's originals vault.

Normally Restore protects artwork changed outside Overcoat. Enable this only when the vaulted copy should win regardless.

Recovery

Overcoat keeps a clean copy of every poster before it overlays it. This is what Restore original posters puts back, and it survives uninstalling the plugin — so a forgotten restore is recoverable by reinstalling and running it then.

Checking…

These figures are read when the page opens. Recheck reads them again — useful after a run, a restore, or removing something from your library. It only looks; it doesn't change any posters.
If something looks wrong
  • One poster is wrong. In Jellyfin, open the item → Edit images, or refresh its metadata with Replace existing images. Overcoat will re-overlay it on the next run.
  • You want your original posters back. Run Restore original posters above. It skips anything whose art was changed outside Overcoat, so it won't overwrite work you did yourself.
  • You uninstalled without restoring. Reinstall Overcoat and run Restore — the saved copies are kept outside the plugin folder and are still there.
  • No saved copy exists (the count above is not zero). Restore cannot help those. Refresh the library's metadata in Jellyfin with Replace existing images to pull fresh artwork from your metadata providers.
